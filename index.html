为什么需要

一个产品从 0 到 1，不是一个 AI 能搞定的

角色专业化

PM 负责需求，Architect 负责架构，Developer 负责实现——每个 Agent 只做自己擅长的事，不跨界。

协同自动化

Agent 之间自动 Handoff：PRD→架构→设计→开发→评审→部署，流水线式推进。

开源可审查

16 个 Agent 模板开源在 GitHub，每个角色的 system prompt 都可阅读、可修改、可贡献。

协同流程

从想法到上线，流水线式推进

战略层：人 + 主 Agent 讨论方向

产品定位、关键取舍由你拍板，Agent 辅助分析

执行层：PM、架构、设计并行启动

产品经理出 PRD，架构师出技术方案，设计师出设计系统

规划层：任务分解

规划师将 PRD 分解为可执行的 Sprint 计划

开发循环：实现 → 评审 → 验证

executor 写代码，code-reviewer 审查，verifier 验证。最多 3 次修复循环。

交付层：部署上线

DevOps 配置 CI/CD，安全审查过合规
